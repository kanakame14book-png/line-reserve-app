# 各ファイルの役割

主要ファイルごとに、担当する責務・主な処理・関連する状態やデータを説明します。

---

## 画面（app/）

### `app/layout.tsx`
- 全ページ共通のルートレイアウト。`<html>` / `<body>` の骨格と Geist フォントを適用する。
- `metadata`（タブタイトル・説明）を定義（現状は `create-next-app` の初期値のまま。Issue #47 で修正予定）。

### `app/page.tsx` — 受験生向け 登録/予約フォーム（`/`）
アプリの入口。LINEミニアプリ（LIFF）内で開かれる想定。
- **LIFF初期化**：`liff.init()` でログイン確認、プロフィール（表示名・userId）取得。
- **モード自動切り替え**：選択した「学部・学科・入試区分」から `data/options.ts` の合格発表日を引き、
  - 発表前 → **登録モード**（`isOfficial=false`／status=`登録`）：志望情報＋アンケートのみ。
  - 発表後 → **予約モード**（`isOfficial=true`／status=`予約`）：来場枠選択＋氏名・連絡先入力。
- **既存データの読み込み**：同一 `line_user_id` の予約があれば読み込み、フォームに反映。
- **入力バリデーション**：ふりがな（ひらがな）・電話番号・メールの形式チェック、満席チェック。
- **送信（`handleReserve`）**：`reservations` テーブルへ upsert（`line_user_id` で競合解決）。予約確定時は LINE トークへ受付票URLを送信。
- **キャンセル（`handleCancelReservation`）**：自分の予約を削除し、LINEにキャンセル通知。

### `app/admin/page.tsx` — 管理画面（`/admin`、要ログイン）
スタッフ用の中枢。Supabase Auth（メール／パスワード）でログイン。3タブ構成。
- **ユーザー管理タブ**：予約一覧の表示・絞り込み（状態／学部／日程）・並べ替え、班・人数・受付状態の編集、受付票を開く。
- **枠管理タブ**：予約枠（`slots`）の作成・編集・削除（`handleSaveSlot` / `handleEditClick` / `handleDeleteSlot`）。
- **受付タブ**：当日の受付状況の管理。
- **班分け（`handleAssignGroups`）**：指定枠の予約者を学部ごとに学科順で並べ、A班・B班・C班へ均等に自動割り当て。
- 主なデータ取得：`fetchReservations()` / `fetchSlots()`。認証状態は `onAuthStateChange` で監視。

### `app/admin/scanner/page.tsx` — QRコード受付（`/admin/scanner`）
- カメラを起動し（`@yudiel/react-qr-scanner`）、受付票QRを読み取る。内カメ／外カメ切替あり。
- 読み取った ID で `reservations` を検索し、状態に応じて分岐：
  - `受付済` → 「受付済み」表示（成功音）。
  - `登録`（予約未確定）→ 受付不可表示（エラー音）。
  - それ以外（`予約`）→ status を `受付済` に更新（成功音）。
  - データなし／不正 → 無効表示（エラー音）。
- **効果音**：`public/sounds/success.mp3` / `error.mp3` を `HTMLAudioElement` で再生（Issue #48）。差し替えは同名mp3の上書きで可能。
- 要管理者ログイン。

### `app/admin/ticket/page.tsx` — 受付票の表示・印刷（`/admin/ticket?id=...`）
- `id` の予約情報を表示し、当日提示用のQRコードを生成。
  - QRに埋め込む値は予約ID単体。`scanner` がカメラで読み取り、このIDから予約を特定して受付する。
- 予約確定者（`予約`／`受付済`）は予約日時・形式・人数を表示。未確定（`登録`）は「合格発表前」の案内を表示。
- 印刷／PDF保存ボタンあり。QRコード生成は外部API（`api.qrserver.com`）を利用。

### `app/components/QrTrademark.tsx`
- 「QRコード」がデンソーウェーブの登録商標であることを示す注記（Issue #25）。QRを扱う画面下部に配置。

### `app/lib/close.ts`
- `tryCloseWindow()`：`window.close()` を試み、閉じられない環境（LINE内ブラウザ等）では案内用コールバックを呼ぶ。

### `app/globals.css`
- Tailwind の読み込みと、ライト／ダークの配色・基本フォントを定義。

## データ・設定

### `data/options.ts`
選択肢マスタと合格発表日、共通の型定義を集約。
- `PREFECTURES`：都道府県一覧。
- `FACULTY_DEPARTMENT_MAP` / `FACULTIES`：学部と学科・コースの対応。
- `BASE_ADMISSION_TYPES` / `MEDICINE_*_ADMISSION_TYPES`：入試区分（医学部は学科別に専用リスト）。
- `MOTIVATION_LEVELS`：志望度の選択肢。
- `DETAILED_ANNOUNCEMENT_DATES`：合格発表日マスタ。フォームの登録／予約モード切替の判定に使う。
  - **注意**：一部に `//for test` の動作確認用日付が混在（例：工学部・前期を2026年に前倒し）。本番前に要見直し。
- `Slot` / `Reservation`：共通で使う型定義。

### `supabase.ts`
- 環境変数（`NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`）から Supabase クライアントを生成し export。鍵が未設定ならビルド時にエラーで停止する安全弁付き。
