# ファイル構成

`line-reserve-app` のディレクトリ・ファイル構成です。`node_modules` などの自動生成物は除いています。

```
line-reserve-app/
├── app/                          # Next.js App Router のルート（画面と機能の本体）
│   ├── layout.tsx                # 全ページ共通のレイアウト（HTML骨格・フォント）
│   ├── page.tsx                  # 受験生向け 登録/予約フォーム（LIFF内で起動）
│   ├── globals.css               # 全体のスタイル（Tailwind読み込み・配色）
│   ├── favicon.ico               # ファビコン
│   │
│   ├── admin/                    # 管理・スタッフ用の画面群
│   │   ├── page.tsx              # 管理画面（要ログイン）予約一覧・枠管理・受付・班分け
│   │   ├── scanner/
│   │   │   └── page.tsx          # QRコード受付（カメラでスキャン）
│   │   ├── checkin/
│   │   │   └── page.tsx          # 受付票URLを開いたときの受付処理
│   │   └── ticket/
│   │       └── page.tsx          # 受付票（QRコード）の表示・印刷
│   │
│   ├── components/
│   │   └── QrTrademark.tsx       # 「QRコード」登録商標の注記コンポーネント
│   │
│   └── lib/
│       └── close.ts             # ウィンドウを閉じる共通処理（閉じられない環境の案内付き）
│
├── data/
│   └── options.ts               # マスタ定義（学部・学科・入試区分・合格発表日・型定義）
│
├── public/                       # 静的ファイル（ビルドせずそのまま配信）
│   ├── sounds/
│   │   ├── success.mp3           # QR受付 成功音（ピロリン♪）
│   │   └── error.mp3             # QR受付 失敗/未予約音（ブブッ）
│   ├── file.svg / globe.svg / next.svg / vercel.svg / window.svg  # テンプレ由来のアイコン
│
├── docs/                         # 引き継ぎ用ドキュメント（本ディレクトリ）
│   ├── 設計書.md
│   ├── ファイル構成.md
│   └── 各ファイルの役割.md
│
├── supabase.ts                   # Supabaseクライアントの初期化
├── README.md                     # プロジェクト概要・セットアップ手順
├── AGENTS.md / CLAUDE.md         # AIエージェント向けの作業ルール
│
├── package.json                  # 依存関係・npmスクリプト
├── package-lock.json             # 依存関係のロックファイル
├── tsconfig.json                 # TypeScript設定
├── next.config.ts                # Next.js設定
├── eslint.config.mjs             # ESLint設定
├── postcss.config.mjs            # PostCSS設定（Tailwind用）
└── .gitignore
```

## 補足

- **画面（ページ）は `app/` 配下のフォルダ構成がそのまま URL パスになります**（App Router の規約）。
  - `app/page.tsx` → `/`
  - `app/admin/page.tsx` → `/admin`
  - `app/admin/scanner/page.tsx` → `/admin/scanner`
  - `app/admin/checkin/page.tsx` → `/admin/checkin`
  - `app/admin/ticket/page.tsx` → `/admin/ticket`
- すべての画面コンポーネントは `"use client"`（クライアントコンポーネント）です。ブラウザ側で Supabase / LIFF と直接通信します。
- サーバー専用のコード（API Route 等）は現状ありません。データアクセスはブラウザから Supabase へ直接行っています。
