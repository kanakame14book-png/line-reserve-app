# オープンキャンパス クーポン運用手順

オープンキャンパス来場者に「事前登録するとその日限定のクーポン（生協店舗で使えるドリンク割引）がもらえる」施策の運用メモです。**LINE公式アカウントの設定（OA Manager）とコードの両方が関わる**ため、片方だけ変えるとズレます。担当者が変わっても再現できるよう、設定内容と手順をまとめます。

---

## 全体像

1. 高校生が会場で **LINE公式アカウントを友だち追加** → あいさつメッセージが届く。
2. あいさつ内のリンクから **事前登録フォーム（LIFF）** を開いて登録。
3. 登録完了時、**コードがその日のクーポンURLを自動でトークに送信**（開催日のみ）。
4. 高校生はリンクからクーポンを受け取り、**当日16:00までに生協店舗で利用**。

- 事前登録フォーム（LIFF）: `https://liff.line.me/2010635082-iQHjYAvx`
- クーポン配布は **事前登録（status「登録」）限定**。予約（合格発表後）には付かない。

---

## 開催日（2026年）

| 回 | 日付 | クーポン有効 |
| --- | --- | --- |
| 1日目 | 2026-08-01 | 当日 16:00 まで |
| 2日目 | 2026-08-06 | 当日 16:00 まで |
| 3日目 | 2026-08-07 | 当日 16:00 まで |

※ 日付は飛び石（8/2〜8/5 は非開催）。この3日だけが「開催日」。

---

## コード側（`data/options.ts`）

日付ごとのクーポンURLを `data/options.ts` の `OPEN_CAMPUS_COUPONS` に定義（**実URLはコードが唯一の正**。二重管理を避けるためこの文書には転記しない）。`getTodayCouponUrl()` が **JSTの当日日付** に対応するURLを返し、開催日以外は空文字を返す（＝クーポン行を付けない）。`app/page.tsx` の事前登録完了メッセージで呼ばれる。

```ts
// data/options.ts（構造のみ。実際のURLはファイルを参照）
export const OPEN_CAMPUS_COUPONS: Record<string, string> = {
  '2026-08-01': '（1日目のクーポンURL）',
  '2026-08-06': '（2日目のクーポンURL）',
  '2026-08-07': '（3日目のクーポンURL）',
};
```

- **日付やURLを変える場合は `data/options.ts` を編集**（要デプロイ）。
- 日付キーは `YYYY-MM-DD`。判定は Asia/Tokyo 固定（端末タイムゾーン非依存）。

## OA Manager側の設定

### 1. クーポン（3枚・日ごと）

各開催日に1枚ずつ、以下の設定で作成する。作成後の「シェア → リンクをコピー」で得たURLを、`data/options.ts` の `OPEN_CAMPUS_COUPONS` の対応する日付に貼る。

| 項目 | 設定 |
| --- | --- |
| 内容 | 生協店舗でドリンク割引（20円引き） |
| 有効期間 | その開催日の開始 〜 **当日16:00まで** |
| 使用可能回数 | **お一人様1回**（再登録による二重取り防止はこの設定に依存） |
| 抽選 | **オフ**（全員がもらえる） |

### 2. あいさつメッセージ（友だち追加時）

あいさつメッセージは**静的**（全員・毎日同じ）で、日付やLIFFの登録内容で自動的に文面を変えられない。そのため **開催日だけ手動で②に差し替える運用**とする。`{友だちの表示名}` はOA Managerの「友だちの表示名」タグを挿入する。

## 差し替えスケジュール

| 日 | あいさつメッセージ |
| --- | --- |
| 8/1（開催）| **② に差し替え** |
| 8/2〜8/5（非開催）| ① に戻す |
| 8/6（開催）| **② に差し替え** |
| 8/7（開催）| **② に差し替え** |
| 8/8 以降 | ① に戻す |

- 差し替えは**その日の朝**、戻すのは**その日の夜〜翌朝**。
- ②の「本日」は読んだ人の当日を指すので、各開催日で文面はそのまま正しく機能する（日付を書き換える必要なし）。
- **②を出す日は、コードの `OPEN_CAMPUS_COUPONS` の日付と必ず一致させる。** ズレると「クーポンをお送りします」と言ったのにコードは送らない（またはその逆）という食い違いが起きる。

## よくある疑問

- **登録を取り消して再登録したら二重取りできる？** → コードは登録のたびにリンクを送るが、実際に使える回数はクーポンの「使用可能回数＝1回」で制限される。二重取りは不可。
- **開催日以外に事前登録した人は？** → コードが空文字を返すのでクーポン行は付かない。あいさつも①なのでクーポンの案内はしない。
- **来年も使う場合は？** → OA Managerでクーポンを作り直し、`OPEN_CAMPUS_COUPONS` の日付・URLを差し替え、あいさつ②の差し替え日を新日程に合わせる。
